# Chronos

个人心理健康洞察系统。从纵向日记文本出发，融合多源语境数据，自动识别情绪锚点、促进与损害因素，生成有证据支持的结构化洞察报告。

**完整项目文档（唯一权威）** → **[docs/project.md](docs/project.md)**

## 功能

### 一期（日记文本）

- 日记导入（Echo entries、JSON、txt，仅当年）
- 形态分类与信息单元抽取（Ollama LLM + 启发式兜底）
- 六类锚点涌现检测
- 情绪时间序列与稳定性分析
- 促进/损害/伪促进因素识别
- 关系网络、语言模式、主题生命周期分析

### 二期（多源语境）

- **设置**：常驻城市配置 → Open-Meteo 历史天气回溯；支持**天气连接测试**
- **时间节律**：星期、节假日、节气（本地计算）
- **可选导入**：Apple Health XML、可穿戴 CSV（**列映射**）、屏幕时间 CSV（列映射）、GPX 轨迹、**手动地点 JSON/表单**
- **语境对齐**：按日合并多源数据，缺失不插补
- **控制变量因素分析**：statsmodels OLS 控制 weather/sleep/weekday，更新 effectSize 与 controlledFor
- **交互效应**：风险/保护组合检测
- **环境敏感性**：天气系数、恢复性/压力空间
- **生理-心理耦合**：睡眠/步数/心率滞后相关
- **个人预警模式**：情绪低谷前信号组合
- 报告新增「环境敏感性」「个人预警模式」板块 + 数据完整性说明

### 三期（叙事构建）

- **锚点关联链**：因果 / 主题 / 人物 / 对照 / 演变五类 chainLinks
- **生命故事书**：从关联链梳理叙事线，可接受/拒绝/标注（`/story`）
- **多元自我**：自我星盘 + 对话河流 + 声音转换模式（`/selves`）
- **叙事重构**：识别内化问题叙事，引导性对话探索替代故事（`/reframe`）
- 报告新增「叙事脉络」板块，可跳转三期视图

## 技术栈

- **桌面端**：Electron 33 + React 18 + TypeScript + Vite
- **分析引擎**：Python 3.11+（FastAPI、pandas、scipy、networkx、jieba、chinese-calendar）
- **天气**：Open-Meteo Archive API（仅坐标+日期，不含日记）
- **语义抽取**：本地 Ollama（默认 `gemma3:4b` 批量抽取；UI 可改）
- **数据**：本地 `data/` 目录（JSON 文件）

## 环境要求

- Node.js 18+
- Python 3.11+
- 网络（仅拉取天气元数据时需要）
- （推荐）Ollama 本地运行

## 快速开始

```powershell
cd F:\commercial\Chronos
.\start.ps1
```

## 使用流程

1. **导入**：同步 Echo 今年日记
2. **设置**：配置常驻城市（如「洛阳」），可点击「测试天气连接」验证 Open-Meteo
3. **数据源**（可选）：导入 Apple Health / CSV（非标准列名时映射）/ GPX / 手动地点
4. **分析**：运行 18 步三期流水线
5. **报告**：查看多板块结论 + 数据完整性
6. **故事 / 自我 / 重构**：翻阅生命故事、查看多元自我、发起叙事重构对话

## 数据目录

```
data/
├── settings.json      # 常驻城市与坐标
├── entries/           # 当年日记
├── context/
│   ├── weather/       # 按日天气缓存
│   ├── wearable/
│   ├── digital/
│   └── location/
├── imports/           # 原始导入文件
├── analysis/runs/     # 分析产物
└── meta/
```

## 隐私原则

- 日记原文在本地处理，不上传云端
- 天气 API 仅发送经纬度与日期范围
- 分析结论区分「原文明说」与「系统推断」
- 缺失数据不插补、不猜测
